adb.exe 취약점 공격을 막기 위해 더미파일로 교체 후 수정하지 못하도록 권한설정하고, 로컬 보안 정책의 소프트웨어 제한 정책 - 추가 규칙에서 adb.exe를 아예 실행하지 못하도록 보안 규칙을 추가했음. LDPlayer를 업데이트 시, adb.exe 파일이 수정되는 과정이 있는 것 같으므로, adb.exe의 더미파일에 대한 권한 설정만 허용해주면 될 것 같음. 업데이트 완료 후에는 다시 더미 파일로 교체하여 수정하지 못하도록 권한설정할 것. 권한 설정 방법 속성 - 보안 - 편집 들어가서 모든 그룹/사용자의 권한을 거부로 설정함. 단, 읽기 권한까지 거부하면 LDPlayer가 멈추는 경우가 발생한다고 함. 따라서, 추가적으로 일부 권한만 허용하려면 보안 탭으로 돌아가서 고급 버튼을 클릭. 유형이 거부인 항목들을 편집하면 된다. 고급 권한 표시를 눌러서 거부하지 않을 항목들을 체크 해제해준다. 유형이 "거부" 이므로, 체크된 권한을 거부하고 체크되지 않은 권한은 거부하지 않는다. 유형은 허용과 거부가 있는데, 특정 권한에 대한 허용 설정이 있으며 해당 권한에 대한 거부 설정이 존재하지 않는다면 해당 권한은 허용된다. 특정 권한에 대한 허용 설정이 있으며 해당 권한에 대한 거부 설정이 존재한다면 해당 권한은 거부된다. 특정 권한에 대한 허용 설정이 없다면 해당 권한은 거부된다. 즉, 거부 설정은 허용 설정보다 우선한다. 애초에 허용되지 않은 권한은 거부 설정이 딱히 없더라도 거부된다.
